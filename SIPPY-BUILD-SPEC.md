# SIPPY — Google Play MVP Build Spec for Claude Code
**Package:** `com.genartstudios.sippy` · **App name:** SIPPY · **Tagline:** *one more sip.*

This document is the single source of truth for building SIPPY as a shippable Android app.
Hand it to Claude Code at the repo root. Work milestone by milestone; do not skip ahead.

---

## 1. Architecture decision (locked)

**Capacitor 7 + static web app (vanilla JS, no bundler) + one local Capacitor plugin for Play Games Services v2.**

Why: the game is already a self-contained HTML5 canvas/Web Audio app (see `reference/sippy-prototype.html`).
Capacitor wraps it in a native shell and provides maintained plugin rails for AdMob, Firebase
Analytics, and IAP. Play Games Services v2 has no maintained community plugin, so we write a
small local one in Kotlin (~150 lines). No React/Vite/TypeScript build chain — `www/` is served
as static files. Rude and crude, by design.

**SDK targets (locked by Play policy, June 2026):**
- `targetSdk = 36` (Android 16 — required for new apps from Aug 31, 2026; ship future-proof now)
- `minSdk = 23` (Android 6.0 — ~99% of active devices; WebView is Play-updated so even old
  devices get a modern canvas/Web Audio engine)
- Release artifact: signed `.aab` (App Bundle), Play App Signing enabled
- Play Billing: must be on Billing Library v7+ (RevenueCat plugin satisfies this; verify at build time)

## 3. CLAUDE.md (place at repo root, verbatim)

```markdown
# SIPPY — working agreement
- The game logic lives in www/js/. It must run in a plain desktop browser with zero native
  plugins (native.js provides no-op fallbacks). Test in browser FIRST, device second.
- All gameplay tuning goes in balance.js. Never scatter magic numbers.
- All native access goes through native.js. No plugin imports anywhere else in game code.
- No localStorage / sessionStorage anywhere. Use save.js (Capacitor Preferences).
- Audio: dynamic sounds (buzz, slurp, heartbeat) are synthesized; one-shots are .ogg samples
  with synth fallback if a file fails to load.
- Commands:
  - dev in browser:  npx serve www
  - sync native:     npx cap sync android
  - debug on device: cd android && ./gradlew installDebug
  - release bundle:  cd android && ./gradlew bundleRelease
- Never commit: google-services.json, *.keystore, RevenueCat keys (use local.properties / env).
- Before claiming a milestone done, run its acceptance checks (spec §10).
```

## 5. Feature wiring (game ↔ native)

**Rewarded ad — "Second Wind":** on splat, before game over, offer once per night:
*"Sippy's ghost refuses to leave. Watch an ad to un-splat (keeps your banked mL, current sip is lost)."*
`native.showRewarded()` → on reward: resume at next giant. On no-fill/decline: straight to game over.
Frequency cap: max 3/night. No banners, no interstitials — protect the suspense.

**IAP (RevenueCat offerings → Play products, all non-consumable):**
- `remove_ads_999` — removes the rewarded prompt's ad requirement (Second Wind becomes free, 1/night)
- `hat_pack_party_199` — party hat, propeller cap, tiny sombrero
- `hat_pack_fancy_199` — top hat, crown, tiny crocs (worn on two of his six legs)
Hats are drawn procedurally in `hats.js` as canvas functions anchored to Sippy's head transform —
no image assets. Hat shop = DOM overlay on title screen; equipped hat persists via save.js
and appears in death/ghost/share scenes (the ghost keeps the hat — important for comedy).

**Play Games Services v2:**
- Leaderboards: `best_night_ml` (higher=better), `closest_call_ms` (lower=better)
- Achievements: `first_splat` (get slapped), `clutch` (first windup escape),
  `frame_perfect` (escape ≤0.05s), `decathlon` (10 giants in one night),
  `bloodbank` (1,000 mL lifetime), `amateur_hour` (splat on Night 1),
  `coward` (escape 10 sips under 10 mL — "playing it disgustingly safe")
- v2 auto-signs-in; never block gameplay on sign-in failure. IDs come from Play Console
  (Grow → Play Games Services) and land in `strings.xml` + a `gameids.js` constants file.

**Firebase Analytics event taxonomy (keep it to exactly these):**
`sip_end {ml, clutch, night, giant_type}` · `splat {night, lost_ml, hold_s}` ·
`night_end {banked_ml, giants, closest_ms}` · `second_wind {source: ad|iap, night}` ·
`iap_purchase {sku}` · `hat_equipped {hat_id}`

## 6. ElevenLabs MCP — sound manifest

Generate via the ElevenLabs MCP (sound-effects endpoint for SFX, TTS for voice lines), then run
`tools/normalize_audio.sh` (ffmpeg → mono ogg, ~-14 LUFS, trim silence, each ≤100 KB).
Audition each against the synth placeholder; keep whichever is funnier.

| File | Type | Prompt |
|---|---|---|
| slap.ogg | SFX | "sharp cartoon open-palm slap on skin, punchy, comedic, very short" |
| splat.ogg | SFX | "wet cartoon squish splat, small bug, comedic, short" |
| snore_in.ogg | SFX | "deep rumbling cartoon snore inhale, large sleeping man, 0.7 seconds" |
| snore_out.ogg | SFX | "soft whistling snore exhale, comedic, 0.5 seconds" |
| gasp.ogg | SFX | "sharp surprised waking gasp, large man, comedic, short" |
| jingle.ogg | SFX | "tiny cheerful victory jingle, toy xylophone, 3 ascending notes, 1 second" |
| clutch.ogg | SFX | "triumphant tiny fanfare, kazoo and xylophone, comedic, 1.5 seconds" |
| harp.ogg | SFX | "small angelic harp glissando descending, comedic heaven, 2 seconds" |
| vox_onemoresip.ogg | TTS | tiny squeaky voice: "one more sip…" (pitch up ~+5 st in normalize step) |
| vox_uhoh.ogg | TTS | same voice: "uh oh." |
| vox_worthit.ogg | TTS | same voice: "worth it." (plays over the ghost, 30% chance) |

Voice lines are SIPPY's brainrot hook — the TikTok sound is `vox_onemoresip` + slap + harp.
Dynamic audio (buzz weight, slurp pitch, heartbeat tempo) remains WebAudio synthesis — it must
track gameplay continuously and samples can't.

## 7. Native shell specifics

- `MainActivity.kt`: edge-to-edge immersive (hide status/nav via WindowInsetsController),
  `FLAG_KEEP_SCREEN_ON`, hardware-accelerated WebView (default), back button = pause overlay.
- Manifest: `android:screenOrientation="portrait"`, AdMob app ID meta-data, Games APP_ID
  meta-data, `android:isGame="true"`, no INTERNET-optional trickery — INTERNET + AD_ID perms
  come with the ad SDK and must be declared honestly in Data Safety.
- Icons: `tools/make_assets.py` renders Sippy's face (the prototype's drawing math, ported to
  Pillow) on the night-purple `#241647` ground — adaptive foreground/background layers, all
  densities, plus 512 store icon and 1024×500 feature graphic (Sippy + "one more sip.").

## 8. Consent & compliance wiring

- **UMP consent:** call AdMob's consent flow on first launch before any ad request (EEA/UK
  requirement). If consent unavailable → non-personalized ads. Settings overlay gets a
  "Privacy options" button that re-opens the form (required).
- **Target audience (Play Console):** declare 13+ / not directed at children. The art is cute
  but the design (push-your-luck + ads) should not enter Families policy scope.
- **Content rating questionnaire:** mild cartoon/fantasy violence (comedic slap), no blood
  *of the human variety shown* — Sippy's fill is stylized; answer the violence questions as
  cartoon violence. Expect Everyone 10+ / PEGI 7 ballpark.
- **App Set/Advertising ID:** declared via the ad SDK; Data Safety must list it (see §9).

## 9. Data Safety form (exact stance)

Data collected: Device or other IDs (Advertising ID) — collected, shared with ad partners,
for advertising; App interactions (analytics events) — collected, not shared, for analytics;
Purchase history — collected (Play Billing/RevenueCat), for app functionality.
No location, no contacts, no personal info fields. Data not sold. Encryption in transit: yes.
Deletion: state that gameplay data is device-local and removable by uninstall; analytics
retention per Firebase defaults. Mirror all of this in `store/privacy-policy.md`.

## 10. Milestones (one Claude Code session each, with acceptance gates)

1. **Port.** Split prototype into `www/` modules per §2. Gate: full game loop playable in a
   desktop browser via `npx serve www`, with native.js stubs logging calls.
2. **Shell.** `npx cap add android`, MainActivity, manifest, icons/splash from make_assets.py,
   haptics wired. Gate: `./gradlew installDebug` runs on a phone; slap vibrates; portrait
   immersive; 60fps on a mid-range device.
3. **Sound.** ElevenLabs MCP generation per §6, normalize, hybrid audio.js. Gate: all files
   ≤100 KB, synth fallback works in airplane mode with files deleted.
4. **Ads.** AdMob plugin + UMP + Second Wind flow with frequency cap, using Google's TEST ad
   units until release config. Gate: revive works; decline path clean; consent form appears
   on a fresh install with EEA test geography.
5. **IAP.** RevenueCat + hat shop + remove_ads behavior + restore purchases button. Gate:
   sandbox purchase of each SKU; hats render on Sippy, ghost, and share card.
6. **Play Games.** Local plugin; auto sign-in; scores submit on night_end; achievements fire.
   Gate: leaderboard UI opens from title; all 7 achievements unlockable; sign-in failure
   never blocks play.
7. **Ship prep.** Firebase analytics events, versionCode 1, signed `bundleRelease`, store/
   docs finalized. Gate: `bundletool` installs the AAB locally; release-checklist.md all green.
